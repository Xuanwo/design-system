One Operator,
every backend.

One Operator handle hides every difference between S3, GCS and local disk.

Why another abstraction?

Same shape

Read, write, list, stat — identical across twenty services. Learn once, use everywhere.

Honest cost

The abstraction hides complexity, not cost. Every call maps to one request.

Composable

Retry, logging, metrics, limits — added as layers, never forked code.

Before & after

Before
  • One SDK per provider
  • Different errors, retries, auth
  • Rewrite to switch backends
After
  • One Operator API
  • Uniform errors & behavior
  • Swap backend in one line

A good abstraction doesn’t hide the cost — it makes the cost predictable.
Keep the complexity in the library; hand the simplicity to the user.

Read a file — in five lines

src/main.rs rust 
use opendal::Operator;

async fn read(op: Operator) -> Result<Bytes> {
    // one call — retries & auth handled below
    let bs = op.read("config.toml").await?;
    Ok(bs)
}
